⚠ Admin PIN Recovery
If admin PIN is forgotten:
1. Go to console.firebase.google.com
2. Select project cts-production-7069c
3. Firestore → cts > state > authConfig > adminPinHash
4. Replace value with -1935285948 (default PIN: 2026)
5. Click Update — done
If admin password is forgotten:
1. Go to console.firebase.google.com
2. Select project cts-production-7069c
3. Firestore → cts > state > systemSettings > adminPwd
4. Read the value — that is your password
5. Or replace with a new password of your choice

index.html — Landing & Login
First page everyone sees. Staff login here, guests view the manual.
All users Guest
▶
Staff login via ☰ menu
Staff tap the hamburger button top-right to open the login drawer. Choose your role — Admin needs PIN + password, Supervisor needs PIN only, Worker selects their name then enters PIN.
Example — Admin login
1Tap ☰ top right → tap Admin
2Enter 4-digit PIN on the keypad
3Enter password → tap Enter
4You are taken to the Dashboard
Guest / View Manual button
Guests (customers, visitors) tap the yellow button to go directly to the Device Manual page without any login. They can view and download device PDFs only. No access to any production data.
Product pills
The pills (Energy Eye GSM, AC Eye, etc.) on the landing page load live from Firestore. They update automatically when you add or remove products in the Inventory page. No code change needed.
dashboard.html — Production Dashboard
Main workspace for all staff. Three tabs: Attendance, Assembly, Charts.
Admin Supervisor Worker
▶
Attendance tab — Clock in / Clock out
Workers tap their card to clock in when they arrive and clock out when they leave. The system records the time and calculates hours worked automatically. Overnight shifts are detected automatically.
Example — Worker clocks in
1Login as Worker → Attendance tab
2Tap Clock In on your card
3Button changes to Clock Out — time is recorded
Attendance tab — Overtime request
Workers can request overtime from their card. Supervisor or Admin must approve or reject it. Approved OT is included in the attendance report and export.
Example — Approve OT request
1Login as Supervisor → Attendance tab
2Find the worker with a pending OT badge
3Tap Approve or Reject
Assembly tab — Update parts progress
Workers update how many units they completed for each assigned part. Supervisor can see overall project progress. Admin can configure parts and targets in the admin settings area.
Example — Worker updates units
1Login as Worker → Assembly tab
2Find your part → enter units completed today
3Tap Update — progress bar refreshes
Assembly tab — Send WhatsApp report
Supervisor or Admin can send the current assembly progress as a WhatsApp message to configured contacts. The message includes all parts, units done, and percentage complete. Contacts are configured in Admin → System Settings.
Charts tab
Visual charts showing attendance patterns and assembly progress over time. Admin and Supervisor see all projects. Workers see their own project only. Useful for weekly and monthly reviews.
admin.html — Admin Panel
Full system management. 6 tabs. Admin login required.
Admin only
▶
Users & Permissions tab
Add or remove Workers and Supervisors. Set which features a Supervisor can access — Attendance Edit, Inventory Edit, PO Management, Reports Export, Manual Page. Changes take effect on the next login.
Example — Add a new worker
1Admin Panel → Users & Permissions
2Scroll down → tap + Add Worker
3Enter name and select project → Save
4Worker can login immediately with default PIN 0000
PIN Management tab
View all users and their last login time. Change any PIN or reset it to the default. Session Reset forces all devices to log out — useful when a device is lost or shared.
Example — Worker forgot PIN
1Admin Panel → PIN Management
2Find the worker → tap Change
3Tap Reset to Default → PIN becomes 0000
4Tell the worker to login with 0000 and change it
System Settings tab
Change the admin password, company name, and system alert thresholds. The admin password is stored securely in Firestore and can be changed here at any time without touching any code.
Manuals tab
Enable or disable device cards on the manual page. Add, edit, or delete technical download links that appear for technical staff.
Example — Enable Energy Eye GSM manual
1Upload manual-ee.pdf to GitHub
2Admin Panel → Manuals → tick Energy Eye GSM
3Card shows Available on manual page instantly
Activity Log tab
Every login, clock-in, PIN change, and admin action is recorded with timestamp and who did it. Filter by type or date. Delete individual entries or reset the entire log.
Example — Check who logged in today
1Admin Panel → Activity Log
2Filter by today's date → filter type: Login
3See all logins with time and name
App Reference tab — this page
Full documentation of all pages, features, and how to use them. Use the Print PDF button to save a copy for offline reference.

View all parts with current stock, minimum level, and status (OK / Low / Out). Use + IN and - OUT buttons to adjust. View the movement log for each part.

Record purchase orders coming in (Inbound) and goods going out to customers (Outbound). Each PO has a date, supplier/customer, reference number, and items. Export as CSV for accounting.
Products tab
Manage the list of finished products (e.g. Energy Eye GSM, Defrost on Demand). These products appear as pills on the landing page automatically — no code change needed.
Export reports
Export Stock Report, Inbound PO, Outbound PO, or Full Report as CSV. Open in Excel or Google Sheets for further analysis and record keeping.
manual.html — Device Manuals
Public device manual index. No login required.
All users Guest

Device cards
Shows a card for each CTS device. Available cards can be clicked to open the PDF. Coming Soon cards are greyed out. Availability is controlled from Admin → Manuals tab in real time.
Example — Open Defrost on Demand manual

A section at the bottom shows download links for technical tools (e.g. CTS Rulez, CTS RulezPRO). Managed by Admin from the Manuals tab. Only visible when links have been added.
Adding a new device manual
Upload the PDF to GitHub with the correct filename (manual-ee.pdf, manual-ac.pdf, manual-fe.pdf), then enable the device from Admin → Manuals tab. The card becomes clickable immediately — no page refresh needed.
Security & Recovery
Lockouts, WhatsApp alerts, and emergency procedures.
Admin
▶
PIN lockout — Admin & Supervisor
After 3 wrong PIN attempts a WhatsApp alert is sent to 60192091913. After 5 wrong attempts the account locks for 15 minutes automatically. The lockout clears itself — no action needed.
WhatsApp alert message
🚨 CTS PRODUCTION SECURITY ALERT
ADMIN PIN failed 3 times.
Time: 18/05/2026, 10:30 am
Please check Admin Panel → Activity Log.
Password lockout — Admin
Same rules apply to the admin password. 3 wrong attempts → WhatsApp alert. 5 wrong attempts → locked 15 minutes. If you forgot the password go to Firebase Console → cts/state → systemSettings → adminPwd to read or replace it.
Worker PIN lockout
After 3 wrong attempts the worker is blocked and a WhatsApp is sent to all contacts in the project's WA Contacts list. Admin resets the PIN from Admin Panel → PIN Management → Change → Reset to Default.
Emergency Reset — Life Jacket URL
If you are locked out and cannot login as Admin, use the emergency URL saved privately on your phone. It resets the Admin PIN back to the default silently. After reset, login with the default PIN and change it immediately from Admin Panel → PIN Management.
Recovery procedure
1Open the emergency URL from your private notes
2Wait for green toast: ✓ Admin PIN reset to default
3Login normally with the default Admin PIN
4Admin Panel → PIN Management → change your PIN immediately
